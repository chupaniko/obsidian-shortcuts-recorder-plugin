# Obsidian Shortcuts Recorder Plugin

Record and manage keyboard shortcuts inside Obsidian.

## Usage

- Place the cursor immediately after a `%` character in any Markdown file to reveal the **Record keyboard shortcut** command from the editor menu. This works even outside Markdown tables.

## Installation

1. Download `manifest.json` and `main.js` directly from this repository (for example, via the GitHub “Download raw file” option). Сompiling the project locally is **not required** for installing the plugin in Obsidian.
2. Copy the downloaded files into your Obsidian vault under `.obsidian/plugins/obsidian-shortcuts-recorder-plugin/`:
   - `manifest.json`
   - `main.js`
   - `styles.css` (если появится в будущих версиях)
3. Reload Obsidian or enable the plugin from **Settings → Community plugins**.

## Development

1. Install dependencies:
   ```bash
   npm install
   ```
2. (Необязательно) Соберите плагин локально:
   ```bash
   npm run build
   ```
   Учтите, что собранный таким образом `main.js` предназначен для разработки и может отличаться от опубликованной версии. Для установки в рабочий Obsidian используйте файлы из репозитория, как описано выше.

## Project Structure

- `manifest.json` – Obsidian plugin manifest with metadata and compatibility.
- `src/main.ts` – Source code for the plugin entry point written in TypeScript.
- `main.js` – Bundled output generated by the build step.

## Manual testing

To validate the shortcut capture flow, exercise the plugin manually in Obsidian 1.5 or newer on:

- macOS (for example, macOS Sonoma 14) – ensure Command/Option-based combinations render with platform-specific symbols such as `⌘`, `⌥`, and `⇧`.
- Windows 11 or a modern Linux distribution – ensure Control/Alt-based combinations render in the `Ctrl +`/`Alt +` format.

**Limitations:** Operating systems and Obsidian reserve some global shortcuts (for example, `Cmd+Tab`, `Ctrl+Alt+Delete`, or `Cmd+Space`). These combinations cannot be intercepted by plugins, so they will not be recordable through the modal.
